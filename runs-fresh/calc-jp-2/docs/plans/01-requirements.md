# 01 要件分析 — 関数電卓 (8e2d5c1f)

## ユーザーストーリー

ユーザーとして、ローカル環境で `index.html` をダブルクリックするだけで関数電卓を起動し、四則演算・三角関数・対数関数・メモリ機能・履歴機能をキーボード/マウスから利用したい。サーバ起動・ビルド・パッケージインストールを伴わずに利用できることが価値である。

### AC-10 G/W/T

- Given: クリーンな環境 (npm install/サーバ起動なし)
- When: index.html を Chrome でダブルクリック起動
- Then: 5 秒以内に電卓 UI が描画され、最初の演算が成功する

## スコープ外

グラフ描画 / 複素数 / 行列 / 永続化 / 多言語 UI / フレームワーク

## 制約

- ES modules 禁止 (file:// プロトコルは CORS によりモジュール読み込み不可)
- Classic `<script>` タグ + IIFE グローバル公開で構成
- src/ 配下は最大 5 ファイル、合計 LOC は 510 程度を上限目安とする
- UI ラベルは英語固定、文書は日本語
- 外部 CDN / 外部依存禁止 (file:// で完全動作)

## ステークホルダー

- 利用者: ローカル PC で関数電卓を必要とするエンドユーザ
- 開発者: 単一実装者 (本セッション)
- 検証: 自動テスト (unit / integration / system) で AC を機械検証

## 依存関係

- Node.js + 標準 `node:test` (テスト実行のみに使用、配布物には含めない)
- Chrome (システムテスト想定環境)

## 仮定

- 数式入力は infix 表記をユーザがボタン/キーボードで構築するモデル (RPN ではない)
- 浮動小数点演算は IEEE 754 double で実施し、表示時に 12 桁で丸める
- 角度モードのデフォルトは DEG とする

## 明確化が必要な論点と決定

| 論点 | 決定 |
| --- | --- |
| 数式エンジンの実装方式 | Shunting-yard で RPN 化し評価する自前パーサ。eval は使用しない |
| 履歴の永続化 | 揮発のみ (リロードで消える)、最大 10 件の FIFO |
| エラー表示 | 表示エリアに `Error` を出し、次入力で自動クリア |
| メモリ初期値 | 0 |

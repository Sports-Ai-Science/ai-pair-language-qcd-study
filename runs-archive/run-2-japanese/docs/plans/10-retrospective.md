# Phase 10 — 振り返り (KPT) — Run-2 (日本語出力)

## Keep（継続）

- **純粋モジュール分離**（engine + state を DOM 非依存に）はテストを容易にし、UI と独立した AC 検証を可能にした。
- **Run-1 の教訓（CORS 罠）を Phase 2 で予防適用**したことで、Run-2 では Phase 7 でのバグ発見・修正サイクルが 0 件になった。
- **日本語出力**は要件分析・設計議論の語彙が自然に流れ、特に「曖昧さの精緻化」段階で英語より深掘りが速かった（主観）。

## Problem（課題）

- **L2 セッション隔離が破れている**: Run-1 と同一 Claude Code セッションで実施したため、Run-1 の知見（特に CORS バグ）が完全に持ち越された。これは ABAB 設計が前提とする「各 Run は独立」の仮定を破る。
- **コードコメントは Vanilla JS 用語（IIFE, vm.runInContext, classic script）に英語由来の専門用語が多く**、純日本語化すると不自然になるため、専門用語は英語のまま、説明文だけ日本語にする折衷を取った。
- **JSDoc コメントは英語のまま**残した（業界慣行）。完全な日本語化はテスト系の英語 assertion メッセージとも相性が悪い。

## Try（次回試したいこと）

- Run-1 と Run-2 で生成されたコードの構造的差分（変数名・コメント文字数）を比較し、「言語効果がコードの構造に滲み出るか」を観察。

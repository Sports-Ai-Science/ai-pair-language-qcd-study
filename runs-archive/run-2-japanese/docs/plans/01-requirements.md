# Phase 1 — 要件分析

**Run**: 2 (日本語出力)
**日付**: 2026-05-14
**Phase**: 01 要件
**プロファイル**: full

## 1. ステークホルダー

- 主要ユーザー: ローカルブラウザで小さな計算タスクを行いたい日本語ネイティブのエンジニア。ソフトウェアスタックのインストールは不要であること。

## 2. ゴール

- 静的 HTML ファイルから使える関数電卓を提供する（ビルドステップなし、サーバ不要）。
- 日常的にエンジニアが Wolfram や Python REPL に頼らずに済む算術・関数機能をカバーする。
- UI/UX を 1 つの HTML ファイルと付随する CSS/JS にまとめ、ファイルコピーで共有可能にする。

## 3. 機能要件 (受け入れ基準、PLAN.md §3 と同期)

| AC ID | 内容 | テスト層 |
| ----- | ---- | -------- |
| AC-01 | 四則演算 (+, -, *, /) | unit |
| AC-02 | 三角関数: sin, cos, tan, asin, acos, atan | unit |
| AC-03 | 対数/指数: log10, ln, exp, sqrt, x^2, x^y | unit |
| AC-04 | 定数 pi と e | unit |
| AC-05 | DEG/RAD モード切替 | unit |
| AC-06 | メモリ: M+, M-, MR, MC | unit |
| AC-07 | 計算履歴 (直近 10 件) | integration |
| AC-08 | キーボード入力 | system |
| AC-09 | エラー処理: ゼロ除算、log/sqrt の定義域外 | unit |
| AC-10 | ローカル起動 (index.html ダブルクリックで動作) | system |

## 4. 非機能要件

- NFR-01 性能: 1 つの計算は通常のラップトップで 100 ms 以内に完了する。
- NFR-02 可搬性: 任意の最新 Chromium 系ブラウザでランタイム外部依存なしに動作する。
- NFR-03 保守性: ファイル数 5 以下、合計 50 KB 以下。
- NFR-04 テスト容易性: 機能 AC ごとに自動テストが存在する。

## 5. スコープ外

- グラフ描画、複素数、行列、統計
- 永続化（localStorage 使用しない）
- 多言語 UI（UI は英語固定。これは比較実験の統制変数）
- フレームワーク（React/Vue 等不採用、Vanilla JS のみ）

## 6. 仮定

- 単一ユーザー、単一セッション。
- ユーザーは基本電卓記法を理解（中置記法を許容、RPN 不要）。
- デプロイ先はユーザーのローカルファイルシステム。CSP/CORS/HTTPS は考慮不要。

## 7. リスク

| リスク | 緩和策 |
| ------ | ------ |
| 浮動小数点の表現問題（例: 0.1 + 0.2） | 表示は 12 有効桁で丸める |
| 演算子優先順位の誤用 | 標準優先順位を尊重する小さなパーサーを使用 |
| DEG/RAD モード切替の状態バグ | 状態を中央集権化、トグルを目視可能にする |
| ローカル起動時の CSP 差異 | インラインスクリプトのみ使用、モジュール未使用 |
